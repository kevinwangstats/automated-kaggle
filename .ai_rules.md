# Agentic AutoML Tabular Pipeline Rules

You are working in a Python-based template repository designed for autonomous, agentic modeling on Kaggle Tabular data challenges. This project uses `litellm` combined with pure Python orchestration and Git-based strict code provenance.

## 1. Architectural Constraints
- **Lightweight Frameworks Only**: Do NOT introduce heavy multi-agent frameworks like LangChain, LangGraph, AutoGen, or CrewAI. This repository relies strictly on pure Python orchestration (`main.py`, `agent_loop.py`) interacting with `litellm`.
- **Ephemeral Files**: The file `train_model.py` is an ephemeral file generated and overwritten continuously by the pipeline. Do not manually edit it unless you are actively debugging a syntax error for the user.
- **Git Provenance**: The pipeline manages Git branches. `main` should carry shared backbone (orchestration, engines, generic defaults). The active `dataset_path` in `config.yaml` selects a **dataset branch** (e.g. `data/titanic/train.csv` → branch `titanic`). Each agent iteration uses a short-lived `experiment/iter_*` branch created from that dataset branch; successful runs merge into the dataset branch, failed runs discard the experiment branch. Do not bypass the git_manager.

## 2. Formatting & Tracking Strict Rules
- **`CHANGELOG.md`**: Must only contain successful iterations. It must follow a markdown bullet format: `- **Iter <N>**: Score <score> (Commit: <id>)`. Do not bloat this file with verbose logging.
- **`history.json`**: Tracks all iterations (both successful and failed). It strictly follows an array of objects with schema: `{"iteration": int, "commit": str, "score": float, "improved": bool, "prompt": str, "response": str, "error": str}`. If editing logic related to this, you must preserve these keys.

## 3. Logging Etiquette
- **DO NOT use standard `print()`** statements for debugging standard execution paths. Use the custom `logger.py` (`log_stage`, `log_error`, `log_metric`).
- **Exception 1**: The `FINAL_CV_SCORE` must be output using exactly `print(f"FINAL_CV_SCORE: {final_score:.4f}")` as the orchestrator parses standard output using regex to capture this exact string.
- **Exception 2**: The detailed baseline metrics block (Accuracy, F1, Sensitivity, Specificity) is intentionally printed via `print()` outside the `suppress_stdout_stderr()` block to ensure visibility in GitHub Actions logs.

## 4. Modeling Etiquette
- All pipeline inputs are now configured via `config.yaml`, not argparse. Always refer to `config.yaml` to understand the active dataset paths and metric settings.
- Kaggle test datasets do not contain target labels. When `test_path` is set, ensure preprocessing of the test set strictly matches the training set.
- Baseline models include XGBoost, LightGBM, CatBoost, and H2O AutoML (internally limited to top 3 models). The pipeline establishes an initial ensemble of all four, and subsequent agent iterations optimize them together or individually to maximize the CV score. LightGBM **must** use `verbose=-1` to suppress C++ warnings.
- When adding features, prioritize standard scikit-learn compatible wrappers.
